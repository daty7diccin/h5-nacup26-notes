# h5-nacup26-notes

## 项目简介

本仓库用于归档和发布多个独立的 HTML 页面。这些页面以静态文件形式存放，不依赖特定域名或后端服务，可直接在浏览器中打开或通过静态托管服务访问。

仓库内容主要涉及各类笔记、工具页、演示页等轻量级前端资源，适合作为个人或小团队的技术资料存档与临时发布渠道。

## 目录结构

```
h5-nacup26-notes/
├── pages/          # 存放所有独立 HTML 页面文件
├── assets/         # 页面引用的公共资源（CSS、JS、图片等）
├── README.md       # 本文件
└── .gitignore      # Git 忽略规则
```

- **pages/**：每个 HTML 文件为一个独立页面，文件名建议反映页面用途或内容主题。
- **assets/**：若多个页面共用样式或脚本，可统一存放于该目录，避免重复。

## 页面归档说明

- 所有页面均为纯前端 HTML，不包含服务器端逻辑。
- 页面间无强依赖关系，可单独使用或分发。
- 归档内容可能随时间更新或清理，不承诺长期保留。

## 维护说明

- 欢迎通过 Issue 或 Pull Request 提交建议或修正。
- 新增页面请放入 `pages/` 目录，并确保引用资源路径正确。
- 若需删除或废弃页面，建议在提交说明中注明原因。
- 仓库维护者保留对内容的最终管理权限。

## 使用方式

1. 克隆或下载本仓库。
2. 直接打开 `pages/` 下的任意 HTML 文件即可预览。
3. 也可将整个仓库部署至任意静态托管服务（如 GitHub Pages、Netlify 等），通过目录结构访问页面。
